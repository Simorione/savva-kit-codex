# savva-kit

**Философия разработки через ИИ, упакованная как переносимый набор.** Метод постоянной
LLM-поддерживаемой вики (по Карпатому) + 4-слойная модель Insomnia + инженерный контур
проверки, обкатанный на реальной ERP/CRM-системе.

Это не библиотека и не фреймворк — это **набор правил, ролей (агентов) и процессов**,
которые заставляют разработку через ИИ оставаться экологичной: без раздувания долга,
без костылей, без «плодящихся сущностей», с проверкой чужими глазами перед закрытием фичи.

Работает поверх любого ИИ-ассистента, который умеет читать инструкции из файлов:
**ChatGPT / Codex, Claude Code, Cursor** и т.п. Ниже — как завести за 5 минут.

> Эта версия — tool-agnostic + готовые Custom GPT-промпты (`prompts-chatgpt/`). Для **Claude Code**
> есть родной вариант (субагенты с YAML-шапками в `.claude/agents/`, скилл, `CLAUDE.md`):
> **https://github.com/Simorione/savva-kit-claude**.

## Быстрый старт (5 минут)

### Вариант A — твой ИИ работает с git-репозиторием (Codex, Cursor, Claude Code)

```bash
git clone <этот-репозиторий> && cd savva-kit
./bootstrap.sh /путь/к/твоему/проекту   # разложит скелет 4-слойки и контракт
```

Дальше открой проект в своём ИИ-ассистенте. Он подхватит `AGENTS.md` (Codex) или
скопированный контракт-файл сам. Первым делом попроси его: **«прочитай контракт проекта
и представься по роли оркестратора»**.

### Вариант B — Custom GPT / ChatGPT-проект (без git)

1. Открой `INSTALL.md` → раздел «Подключение к ChatGPT».
2. Скопируй содержимое `prompts-chatgpt/00-orchestrator.system.md` в **Instructions** своего
   Custom GPT (или в System prompt проекта).
3. Загрузи файлы из `method/` и нужных `agents/` в **Knowledge** проекта.
4. Для роли-специалиста (санитар, аудитор и т.п.) — заведи отдельный чат/GPT с
   соответствующим `prompts-chatgpt/<роль>.system.md`.

## Как это читать по слоям

1. Сначала — `method/00-karpathy-insomnia.md`. Это фундамент: зачем 4 слоя и что нельзя нарушать.
2. Потом — `method/03-review-contour.md` и `method/05-gradation.md`. Это **сердце** философии:
   почему фичу проверяет не тот, кто её писал, и когда гонять тяжёлый контур, а когда нет.
3. `agents/00-orchestrator.md` — как один главный агент дирижирует специалистами.
4. `example-izolation/ERP-CLAUDE.md` — как всё это выглядит вживую на реальном проекте.
5. `templates/project-skeleton/` — заготовка под свой проект: заполняешь плейсхолдеры и начинаешь.

## Плейсхолдеры (заменить под свой проект)

В шаблонной части встречаются:

| Плейсхолдер | Что подставить |
|---|---|
| `{{PROJECT}}` | Название твоего продукта/системы |
| `{{OWNER}}` | Имя владельца проекта (кто ставит задачи и принимает результат) |
| `{{ORCHESTRATOR}}` | Имя главного агента-оркестратора (в примере — «Искин2») |
| `{{STACK}}` | Твой техстек (в примере — Java/Spring + PostgreSQL + Vue 3) |
| `{{WORKSPACE}}` | Корневой путь рабочего пространства проекта |

`bootstrap.sh` умеет проставить их интерактивно.
