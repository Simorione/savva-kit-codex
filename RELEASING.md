# RELEASING — как выкатывать обновление набора

> Этот набор устанавливают **другие люди на других машинах**. Поэтому планка «готово» выше
> обычной: работает не «в моей папке», а «у незнакомого человека после `clone` + одной команды».
> Чек-лист ниже — обязательный к прохождению **буквально по пунктам** перед `git push`.
> Он родился из реальных промахов: заявленные инструменты не доезжали до установки, а скрипт
> после установки уходил в неверный каталог — всё это ловится одним прогоном.

## Definition of Done (перед push)

1. **Прогнать установщик**, а не прочитать. На чистой временной директории, с нуля:
   `./bootstrap.sh /tmp/kit-test`.
2. **Сверить структуру результата** с обещанной в `README.md` (что и куда легло).
3. **Проверить, что пути/ссылки в разложенном виде резолвятся** — `grep` битых путей и
   оставшихся плейсхолдеров (`{{KIT_ROOT}}`, `{{PROJECT}}`…), а не «на глаз».
4. **Запустить КАЖДЫЙ заявленный инструмент из установленного места** — не только проверить,
   что файл лежит:
   - `layer-lint.py` из корня установки — отрабатывает;
   - `freshness-check.sh` из установленного места — находит корень, не падает с «корень не найден»;
   - `pre-commit-guard.sh` — ловит тестовый секрет и большой дифф.
5. **После любого переименования/копирования** — сплошной `grep` старых имён/номеров по всему
   пакету (напр. после `08→07` проверить, что нигде не осталось `08`).
6. **Пройти «путь пользователя» из README дословно:** clone → команда → первый запуск.
7. **Проверить идемпотентность:** повторный `bootstrap.sh` не перезаписывает; `--force` делает
   бэкап.

## Гигиена git/GitHub

- `.gitignore` закрыть **до** первого коммита (артефакты вроде `__pycache__` не должны уезжать).
- `git status` / просмотр staged **перед** `add` — не `git add -A` вслепую.
- **После push** — открыть репозиторий «глазами анонима» (что реально легло, нет ли мусора).
- Токен/секрет для публикации — использовать разово, `remote` оставлять чистым (без токена в URL).

## После выката

- Обновить `CHANGELOG.md` — что добавлено/исправлено, человекочитаемо.
- Если правился формат/номера файлов — синхронизировать документацию и примеры.

> Короткий принцип: **чтение ≠ прогон**. «Файлы разложены» — это не «работает». Пока каждый
> заявленный инструмент не запущен из установленного места, набор не готов к раздаче.
