# Правила распространения изменений между слоями

## Направления потоков

```
inbox/  →  raw/         (если источник — сохраняем навсегда)
inbox/  →  workspace/   (если уже понятно, что станет страницей)
inbox/  →  /dev/null    (если шум, по подтверждению)

raw/         →  workspace/  (через ingest-helper, как draft)
workspace/   →  wiki/       (через wiki-maintainer, по подтверждению)
workspace/   →  /dev/null   (после TTL 14 дней, если не утверждено)

wiki/  ↻  wiki/  (propagate: при изменении страницы — обновить связанные)
```

## Запрещённые направления

- `wiki/ → raw/` — в `raw/` ничего не возвращается.
- `wiki/ → workspace/` — кроме крупного редизайна (сначала draft в workspace, потом замена в wiki).
- `inbox/ → wiki/` напрямую — обязательно проход через `workspace/`.
- Прямые правки в `wiki/` мимо `wiki-maintainer` — запрещены.

## Propagate (обновление связанных страниц)

При изменении страницы X в `wiki/`, `wiki-maintainer`:

1. Находит все страницы, ссылающиеся на X (через `[[wikilinks]]` и `consumers:`).
2. Для каждой связанной страницы Y:
   - изменение **затрагивает цитируемое в Y утверждение** → обновляет Y, метит `updated`;
   - изменение **только добавляет материал** → добавляет в Y ссылку на новый раздел;
   - изменение **противоречит** утверждению в Y → **не правит автоматически**, формирует отчёт
     о конфликте в `workspace/lint-conflict-YYYYMMDD.md`.
3. Обновляет `wiki/index.md`.
4. Делает запись в `wiki/log.md`.

## TTL и устаревание

| Слой | TTL / правило | Что происходит |
|---|---|---|
| `inbox/` | 7 дней | `ingest-helper` готовит отчёт «нерассортированное» |
| `workspace/` | 14 дней | удаление, если не утверждено в `wiki/` |
| `raw/` | бессрочно | не удаляется |
| `wiki/` | по `review_cycle` | если `updated` старше цикла — `status: stale` |

## Обработка противоречий

`wiki-maintainer` **никогда не разрешает противоречия автоматически**:

1. Фиксируется в отчёте `workspace/lint-conflict-YYYYMMDD.md`.
2. Передаётся владельцу на ревью.
3. Решение записывается в `raw/decisions_log.md`.
4. После решения — `wiki-maintainer` применяет его к страницам.
