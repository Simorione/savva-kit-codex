# Contract-first: анти-костыль, анти-дубль

Главная защита от того, чем разработка через ИИ портит кодовую базу: **лишние сущности,
костыли и рассинхрон между фронтом и бэком**. ИИ охотно «досочиняет» недостающее поле,
заглушку, вторую копию логики — и база гниёт.

## Правило

Любая фича, затрагивающая **И фронт, И бэкенд**, начинается с **СОГЛАСОВАННОГО КОНТРАКТА**
(какие данные / поля / эндпоинты нужны), опираясь на существующую модель данных
(`wiki/data-model/`, глоссарий). Только **после** контракта — параллельная реализация.

## Четыре предохранителя

1. **Перед созданием новой сущности/поля — сверка с моделью данных: нет ли уже подходящего?
   Переиспользовать, не плодить.**
   Пример из практики: оплату сертификатом сделали через существующий `PaymentMethod.CERTIFICATE`,
   а не завели новый тип; клиента привязали к существующему `AuthUser`, а не создали параллельную
   сущность.

2. **Фронтенд НЕ выдумывает бэкенд-поля.** Не хватает поля в API — роль-фронтендер **эскалирует
   оркестратору**, а не лепит фейковые данные / заглушку / хранение бизнес-данных в
   localStorage/константах. Это прямой источник рассинхрона.

3. **Оркестратор — интегратор и ревьюер.** Сводит фронт+бэк: проверяет, что обе стороны на
   одном контракте, нет дублирующих сущностей, костылей, рассинхрона типов. Костыль/дубль →
   отправляет переделать, не пропускает.

4. **Семантический долг фиксируется, не зарывается.** Если хак всё же возник ради скорости —
   он записывается в `wiki/recipes/` или бэклог как долг и вычищается позже. Пример: временный
   FIXED-хак для кошелька вынесли в честный `PromocodeType.WALLET`.

## Как это выглядит в потоке

```
задача сквозной фичи
   │
   ├─ оркестратор: сверка с data-model → есть ли уже подходящее?
   │     да → переиспользуем, контракт = существующее
   │     нет → проектируем минимальное новое поле/эндпоинт
   │
   ├─ ФИКСИРУЕМ КОНТРАКТ (поля, типы, эндпоинты)
   │
   ├─ параллельно: инженер (API/логика) ‖ фронтендер (UI по контракту)
   │
   └─ оркестратор сводит: один ли контракт? нет дублей/костылей/рассинхрона типов?
```

## Anti-goal

Не «ИИ быстро сделал, что попросили», а «база после фичи не стала грязнее». Скорость без
этой дисциплины даёт технический долг быстрее, чем человек успевает его замечать.
