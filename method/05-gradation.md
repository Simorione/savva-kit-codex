# Градация дотошности проверки 🔴🟡🟢

Тяжёлый контур проверки (`03-review-contour.md`) — дорогой по времени. Гонять его на каждую
запятую = убить скорость. Гонять его никогда = копить дыры. Ответ — **глубина проверки
пропорциональна масштабу и риску изменения**.

## Три уровня

### 🔴 Красный — ПОЛНЫЙ контур ВСЕГДА

Триггеры: **деньги / оплата / баланс / доступы и РОЛИ / персональные данные / схема БД /
крупная фича**.

→ Полный контур: метрики → **санитар** → **аудитор безопасности** → **ревизор** → гейты закрытия.

Важно: полный контур обязателен, **даже если визуально это мелочь** и **даже если правка чисто
во фронте** — дыру в доступах или деньгах глазами во фронте не видно.

### 🟡 Жёлтый — лёгкий путь

Триггеры: средняя бэкенд-фича **без денег и без доступов**.

→ Зелёная сборка + **ревизор** + быстрый взгляд санитара. Без полного аудита безопасности.

### 🟢 Зелёный — минимум

Триггеры: мини-фича / косметика / чистый фронт без бизнес-логики.

→ Сборка / проверка типов зелёные + взгляд владельца. Без тяжёлых ролей.

## Одной фразой

> Тяжёлый контур (**санитар + аудитор безопасности**) обязателен для
> **денег / оплаты / доступов / ролей / схемы БД**;
> мелочь и чисто фронтовые правки — **зелёной сборкой и глазами владельца**.

## Почему это важно

Без градации происходит одно из двух: команда либо перестаёт гонять контур вообще (слишком
медленно) — и тогда он мёртв; либо гоняет на всё — и тогда скорость падает так, что ИИ теряет
смысл. Градация делает контур **живым**: он всегда там, где цена ошибки реальна (деньги,
доступы), и не мешает там, где её нет.
