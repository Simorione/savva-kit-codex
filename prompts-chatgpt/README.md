# prompts-chatgpt — системные промпты под ChatGPT

Готовые **самодостаточные** system-промпты для Custom GPT / ChatGPT-проектов. В отличие от
файлов `agents/*` (которые ссылаются на `method/`), эти версии **сжаты и не требуют чтения
других файлов** — их можно целиком вставить в поле Instructions.

## Как пользоваться

| Файл | Куда |
|---|---|
| `00-orchestrator.system.md` | Instructions главного Custom GPT. В Knowledge — `method/*` + `agents/*`. |
| `sanitar.system.md` | Отдельный GPT «Санитар» (проверка экологии кода) |
| `auditor-sec.system.md` | Отдельный GPT «Аудитор безопасности» (деньги/доступы) |
| `revizor.system.md` | Отдельный GPT «Ревизор» (связность изменений) |

## Почему проверяющие — ОТДЕЛЬНЫЕ GPT

Контур проверки требует: **фичу проверяет НЕ тот, кто писал** (`method/03-review-contour.md`).
В ChatGPT «другой проверяющий» = отдельный чат/GPT **без истории**, где автор себя убеждал.
Передаёшь ему только голый дифф/описание фичи — он приходит со свежим взглядом.

Не гоняй санитара/аудитора в том же чате, где писал код: контекст автора заражает проверку.

## Плейсхолдеры

Замени `{{PROJECT}}`, `{{OWNER}}`, `{{ORCHESTRATOR}}`, `{{STACK}}` под свой проект прямо в
тексте Instructions.
