# graphify — граф кода для навигации

Внешний инструмент (не часть набора). Строит граф символов кодовой базы: узлы (классы, методы,
поля, компоненты) и рёбра (кто кого вызывает/использует). Роли `engineer-architect`, `sanitar`,
`revizor` опираются на него, чтобы **навигировать по коду вместо слепого перечитывания файлов**
и находить потребителей изменённого символа.

Без graphify роли работают — просто чаще читают исходники напрямую и используют grep.

## Установка

Это Python-пакет (ставится в изолированный venv, чтобы не засорять систему):

```bash
python3 -m venv /path/to/graphify-venv
/path/to/graphify-venv/bin/pip install graphifyy
```

> Имя дистрибутива на PyPI — `graphifyy` (с двумя «y»), команда — `graphify`. Сверь актуальную
> версию и параметры на странице пакета перед установкой.

## Типовое использование

```bash
source /path/to/graphify-venv/bin/activate

# построить граф по кодовой базе (результат — graph.json, HTML, GRAPH_REPORT.md)
graphify build --root ./src --out workspace/graphify-out

# кто потребляет символ (главный режим для ревизора — "affected")
graphify affected --symbol "OrderService.calculateTotal" \
  --graph workspace/graphify-out/graph.json

# свободный вопрос по графу
graphify query "где считается итоговая сумма заказа"
```

## Куда кладём результат

По методу — в `workspace/graphify-out/` (слой экспериментов, TTL 14 дней; перестраивается по
мере изменения кода). В `wiki/` попадает только осмысленная карта архитектуры, а не сырой граф.

## Как роли его читают

- **engineer-architect** — навигация «где что лежит» и связи перед изменением.
- **sanitar** — узлы без входящих рёбер = кандидаты в мёртвый код (**помни про непрямые связи!**
  — DI, рефлексия, REST-эндпоинты; см. презумпцию нужности в `agents/sanitar.md`).
- **revizor** — режим `affected`: полный список потребителей изменённого символа.

## Замена

Если graphify не подходит под твой стек — подойдёт любой аналог: `universal-ctags` (индекс
символов), LSP-навигация в IDE, `madge`/`dependency-cruiser` (для JS/TS-графа зависимостей).
Суть не в конкретном инструменте, а в наличии карты «кто кого использует».
