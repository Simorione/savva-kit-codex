# Гейты закрытия фичи

> Не агенты — **механические проверки**, которые нельзя «уговорить». Прогоняются ПЕРЕД тем, как
> объявить фичу закрытой. По градации дотошности (`method/05-gradation.md`): для 🔴 крупных/
> денежных — обязательны все применимые; для 🟢 мелочи — минимум сборка. Триггер «прогонять
> перед закрытием» канонизируется в конституции проекта.

---

## Гейт A — Зелёная сборка (обязательно ВСЕГДА)

«Фича закрыта» только при зелёной сборке. «Компилируется / тип-чек прошёл» ≠ «работает», но
красная сборка = точно не закрывать.

- **Backend:** команда сборки + тесты проекта — без ошибок.
- **Frontend:** тип-чек + сборка бандла — без ошибок.

Роли обычно и так собирают в smoke — гейт лишь требует, чтобы это было **обязательным**, а не
«забыли».

## Гейт B — Обратимость миграций (для фич со схемой БД)

Каждая новая миграция должна быть **откатываемой без потери данных** и не блокировать таблицу
надолго:

- Под каждым необратимым изменением — явный `-- rollback`.
- Аддитивные DDL (ADD COLUMN nullable, CREATE INDEX) предпочтительны разрушающим.
- Разрушающее изменение (DROP/RENAME) — только через план в две миграции (сначала добавить, потом
  снять после переноса данных).

## Гейт C — Ошибки после стенда

Если полноценный мониторинг ошибок (Sentry и т.п.) на стенде выключен — заменяем его ручной
проверкой лога:

- После деплоя на стенд — глянуть лог сервиса на новые `ERROR`/`Exception`/стек-трейсы, которых
  не было до фичи. Чисто — ок; всплыли новые — разобрать до «закрыто».

## Гейт D — Свежесть опор

Фича не «закрыта», пока опоры не догнали код. Механическая проверка по датам:

```bash
bash tools/gates/freshness-check.sh
```

- **STALE граф** → пересобрать граф кода.
- **STALE аудит-сводка** → контур гонялся не на текущем коде (или не гонялся) — прогнать/обновить.
- **WARN вики** → проверить, канонизирована ли фича (рецепт + решения).

---

## Матрица применимости

| Тип фичи | Гейты |
|---|---|
| 🔴 Крупная / деньги / доступы-роли / схема БД | A + B (если схема) + C + D — ВСЕ применимые |
| 🟡 Средняя бэк-фича без денег | A + (B если миграция) + C |
| 🟢 Мини-фича / косметика / чистый фронт | A (сборка/тип-чек) + взгляд владельца |
